## II. âš™ï¸ Tugas Modifikasi Kode Inti (Wajib)

### A. Tugas 1: Mengubah Nama Executable

**Tujuan**: Mengubah binary yang dihasilkan dari `bat.exe` menjadi `kit.exe`.

**File Target**: `Cargo.toml` (Manifesto proyek Rust, di root folder kit)

**Aksi**:
1. Buka `Cargo.toml`
2. Temukan bagian `[package]` dan ubah nilai key `name` dari `"bat"` menjadi `"kit"`

**Contoh**:
```toml
name = "kit"
```

---

### B. Tugas 2: Modifikasi src/main.rs (Implementasi Fungsi Baru)

**Tujuan**: Menambah logika `touch` dan `mkdir` ke dalam executable `kit` melalui parsing argumen clap.

#### 1. Tambah Imports Fungsionalitas

- Buka `src/main.rs`
- Di bagian atas, tambahkan import untuk library standar Rust yang menangani operasi filesystem

#### 2. Deklarasi Subcommand Baru

- Cari enum yang digunakan untuk menentukan mode operasi (subcommand) (biasanya `Subcommand` atau enum serupa)
- Tambahkan dua case baru untuk mode baru:
  - `CreateFile` (untuk `-c`)
  - `MakeDir` (untuk `-m`)

#### 3. Implementasi Logika Eksekusi

- Cari fungsi utama atau blok `match` besar di `src/main.rs` yang menangani flow program berdasarkan input argumen
- Tambahkan dua case baru ke blok `match` tersebut:

**Untuk CreateFile (`kit -c`)**:
- Ambil nama file dari argumen
- Panggil fungsi Rust yang setara dengan `std::fs::File::create(<filename>)`
- Tambahkan error handling (`match Ok/Err`) untuk memberi tahu pengguna jika file gagal dibuat

**Untuk MakeDir (`kit -m`)**:
- Ambil nama directory dari argumen
- Panggil fungsi Rust yang setara dengan `std::fs::create_dir_all(<dirname>)`
- Tambahkan error handling (`match Ok/Err`) untuk memberi tahu pengguna jika directory gagal dibuat

## III. ğŸ“¦ Tugas Build dan Pengujian

Setelah semua modifikasi kode selesai:

### Kompilasi

Jalankan perintah build Rust di terminal Anda (di root folder kit):

```bash
cargo build --release
```

### Pengujian

Setelah kompilasi selesai, binary `kit.exe` (atau `kit` di Linux/macOS) akan berada di folder `target/release/`. Uji fungsionalitasnya:

- **Uji View**: `target/release/kit src/main.rs`
- **Uji Create**: `target/release/kit -c test.txt`

- **Uji Make**: `target/release/kit -m new_folder/sub_folder`
